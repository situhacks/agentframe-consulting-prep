# Evidence Standards

How to handle claims in a case (and in real consulting work). This is the "audit, don't obey" discipline —
the thing that separates a practitioner from someone who repeats whatever the AI says.
(Distilled from gcamilo/management-consulting — see README for credit.)

## Label every claim

| Type | Symbol | Means | Example |
|---|---|---|---|
| **Fact** | [F] | Observed, sourced, verifiable | "Revenue was $142M in FY24 (10-K p.34)" |
| **Inference** | [I] | Derived from facts via logic | "Growth is decelerating (24%→18%→12%)" |
| **Assumption** | [A] | A gap filled by judgment | "Assume 15% market growth (analyst consensus)" |
| **Estimate** | [E] | Calculated from assumptions + data | "TAM = 500K users × $200 = $100M" |

## Rules
1. Never present an assumption as a fact.
2. Every number needs value + unit + time period + (source, if real). "Revenue $142M" → "Revenue $142M in FY24".
3. State confidence on assumptions: "assume 15% (could be 10–20%)".
4. Say what would change the answer: "if growth < 10%, the NPV turns negative".
5. Prefer base rates to narratives: "3 of 5 similar entries failed" beats "this feels risky".

In a case, you won't have sources — but the *habit* shows: flag what you're assuming, and don't state a guess
as a fact. That's what interviewers (and the Lilli-style live-AI rounds) reward.
