# Frameworks

Lean consulting reference used by the case skills (`run-case`, `case-reviewer`). Three short files — not a
textbook, on purpose:

- **`frameworks-index.md`** — a one-line map of the frameworks worth naming in a case, with when-to-use notes.
  For *picking* the right structure.
- **`case-quality-bar.md`** — the good-vs-bad standard for issue trees and recommendations (MECE-and-specific
  vs canned; answer-first with "so what" + risks + counter-argument vs platitudes). This is what the reviewer
  grades against.
- **`evidence-standards.md`** — the [F]/[I]/[A]/[E] claim-labeling discipline. The "audit, don't obey" habit.

## Why only this much
We deliberately did NOT vendor gcamilo's full framework bodies (~4,400 lines explaining how each framework
works). The reason: a capable model already knows what a profit tree, MECE issue tree, or market sizing IS — it
doesn't need a 781-line reference to remember the definition. What the model *doesn't* reliably have, and what
we kept, is: a consistent **quality bar** (good vs bad), the **evidence discipline**, and a framework **index +
when-to-use**. That's the value-add. The framework depth is learned by *running cases*, not by loading a manual.

If you want the full depth, it's one clone away (see Credit) — drop it in and point the skills at it.
