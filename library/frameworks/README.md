# Frameworks

Lean consulting reference used by the case skills (`run-case`, `case-reviewer`) and `company-research`. Short
files — not a textbook, on purpose:

- **`frameworks-index.md`** — a one-line map of the frameworks worth naming in a case, with when-to-use notes,
  plus case-type playbooks (market entry, M&A, outsourcing, NFP, capacity). For *picking* the right structure.
- **`case-flow.md`** — how a case is *run*: the 3 stages (Open / Analyze / Close) + the habits that score. The
  shared standard the interviewer runs the candidate through and the reviewer grades the flow against.
- **`case-quality-bar.md`** — the good-vs-bad standard for issue trees and recommendations (MECE-and-specific
  vs canned; answer-first with "so what" + risks + counter-argument vs platitudes). This is what the reviewer
  grades the *answer* against.
- **`evidence-standards.md`** — the [F]/[I]/[A]/[E] claim-labeling discipline. The "audit, don't obey" habit.
- **`industries/`** — `_index.md` (16 one-liners) + one file per industry (airlines, CPG, banking, …). Loaded
  one-at-a-time as a *prior* when a case or target firm is in that sector — not as case facts.

## Why only this much
We deliberately did NOT vendor gcamilo's full framework bodies (~4,400 lines explaining how each framework
works). The reason: a capable model already knows what a profit tree, MECE issue tree, or market sizing IS — it
doesn't need a 781-line reference to remember the definition. What the model *doesn't* reliably have, and what
we kept, is: a consistent **quality bar** (good vs bad), the **evidence discipline**, a framework **index +
when-to-use**, the cohort's **case-flow** standard, and **industry priors** the model is unreliable on (specific
profit models, current trends). That's the value-add. The framework depth is learned by *running cases*, not by
loading a manual.

If you want the full depth, it's one clone away (see Credit) — drop it in and point the skills at it.
