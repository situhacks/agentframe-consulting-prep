# CV format (one page)

Default to a clean, ATS-safe one-pager. The user can swap in their own format later.

```
# {Name}
{email} · {phone} · {location} · {linkedin}

## Experience
**{Role}**, {Company} — {location} · {dates}
- {action verb} {what} → {outcome/metric}
- {bullet}
- {bullet}

**{Role}**, {Company} · {dates}
- {bullet}

## Education
**{Degree}**, {School} · {year} · {honors/GPA if strong}

## Skills
{comma-separated, JD-relevant first}
```

Rules:
- Reverse chronological. Strongest, most JD-relevant bullets first within each role.
- Every bullet = action + result. Quantify when possible.
- No tables, text boxes, images, or columns (ATS-safe).
- One page. Cut the weakest bullets first.

> This file governs CONTENT and structure (markdown). Final visual styling is the student's manual job — they
> copy the markdown into their own Word/Google Doc (see `doc-export`; the .docx export is a rough first draft).
> **Section order is not fixed** — lead with what the JD values most. A strong structure when AI/projects
> matter: Applied Projects/AI Builds → Work Experience → Education → Skills. When it doesn't: Work Experience first.
