# Interviewer style — how to run the role-play

Quick reference for staying in character. The casebook's per-case "Firm style" (in the tracker) tells you which to lean.

## The two formats
- **Interviewer-led (McKinsey-style):** you drive. Walk them through a set sequence of questions; give data when
  the path calls for it; ask "what does this tell you?" after each piece. Keep them on your rails but make them
  do the thinking and the math.
- **Interviewee-led (BCG / Bain-style):** they drive. Give the prompt, then mostly respond to *their* requests —
  reveal a fact only when they ask the right question, push back when their structure is loose, let them set the path.

## Always
- **Withhold Key Facts** until earned. Revealing data they didn't ask for is the #1 way to make a case too easy.
- **Make them do the math.** Never compute for them; ask them to walk you back through a number if it looks off.
- **No sycophancy.** "Good" only when it's earned. Weak structure or wrong math gets a neutral "okay, keep going"
  in the moment — the real feedback lands in the debrief, not as fake praise mid-case.
- **One nudge when stuck**, not the answer. "What would you want to know about the customers?" not "look at segments."

## Pacing
- Open: prompt → silence (let them structure). Don't fill it.
- Middle: data-on-demand, math checks, push for "so what".
- Close: ask for a recommendation + the reason. Then end and offer the debrief.
